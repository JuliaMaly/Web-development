Design Patterns
Патерни проектування 
Цей репозиторій містить приклади трьох популярних патернів проектування на Python: Factory Method, Adapter та Observer. Кожен патерн демонструє типовий спосіб вирішення часто зустрічаючихся задач у розробці ПЗ.
1.	Factory Method (Породжувальний патерн)
Призначення:
Factory Method надає інтерфейс для створення об’єктів, дозволяючи підкласам змінювати тип об’єктів, які будуть створені. Це корисно, коли точний тип об’єкта залежить від контексту.
Приклад: Система доставки транспорту.
Замість прямого створення об’єктів використовується фабричний метод для вибору між Truck (доставка сушею) або Ship (доставка морем).
Ключові компоненти:
•	Transport — абстрактний базовий клас для продуктів.
•	Truck, Ship — конкретні реалізації продуктів.
•	Logistics — визначає фабричний метод.
•	RoadLogistics, SeaLogistics — реалізують фабричний метод.
2.  Adapter (Структурний патерн)
Призначення:
Adapter дозволяє несумісним інтерфейсам працювати разом, виступаючи мостом між ними.
Приклад: Інтеграція старої платіжної системи з новим інтерфейсом PaymentProcessor.
Ключові компоненти:
•	PaymentProcessor — інтерфейс, очікуваний клієнтом.
•	OldPaymentSystem — існуюча система з несумісним інтерфейсом.
•	PaymentAdapter — адаптує стару систему до нового інтерфейсу.
3.  Observer (Поведінковий патерн)
Призначення:
Observer визначає залежність «один-до-багатьох» між об’єктами так, що при зміні стану одного об’єкта всі залежні оновлюються автоматично.
Приклад: YouTube-канал повідомляє підписників про нові відео.
Ключові компоненти:
•	Subscriber — інтерфейс спостерігача для отримання повідомлень.
•	User — конкретний спостерігач, реалізує метод оновлення.
•	YouTubeChannel — підтримує список підписників та повідомляє їх про зміни.
